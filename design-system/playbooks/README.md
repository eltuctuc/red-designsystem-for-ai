# Playbooks (framework-agnostic)

These playbooks are short, reusable prompts/instructions intended for an AI working inside a project.

They are **framework-agnostic** on purpose: the DLS provides tokens + rules + contracts + patterns as guidance, but the project chooses its actual rendering stack.

## Files

- `use-tokens.md` – how to include `theme.css` + `base.css` and switch themes
- `generate-component-from-contract.md` – generate a component implementation from a contract
- `generate-screen-from-pattern.md` – generate a screen/flow from a pattern
